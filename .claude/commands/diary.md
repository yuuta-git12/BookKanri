# /diary

## 概要

コマンド実行当日の実装内容や技術的な取り組みをまとめたmarkdownファイルを作成。

## 実行内容

1. **実装履歴の分析**

   - 最近の git コミット履歴を確認
   - 変更されたファイルとその内容を調査

2. **ファイルの内容**

   - 技術的な課題と解決方法
   - 実装のポイントと学び
   - コード例とベストプラクティス
   - 他の開発者への応用可能なナレッジ

3. **ファイルの作成**
   - Markdown 形式で技術記事を生成
   - 実際のコード例を含む実践的な内容
   - `diary/`ディレクトリに保存

## 記事テンプレート

- **タイトル**: 当日の年月日（例：2026-04-01.md）
- **概要**: 何を実装したか、何を学んだか
- **技術スタック**: 使用した技術・ツール
- **実装内容**: 具体的な実装方法とコード例
- **学んだこと**: 失敗や改善点も含めた学び
- **まとめ**: 他の開発者への応用可能性
- **次の予定**: 次の日に実行する予定

## 出力例

``
diary/2026-04-01.md
``

## 使用例

``
/diary
``
