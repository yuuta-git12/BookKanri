# テーブル定義書

## テーブル名

- 論理名：管理区分
- 物理名：permissions
- 説明：管理者の権限区分を管理するテーブル

## テーブル詳細


| No  | PK  | FK  | UK  | カラム名       | 項目名    | データ型      | 桁数  | NOT NULL | デフォルト値 | 列制約            | 備考              |
| --- | --- | --- | --- | ---------- | ------ | --------- | --- | -------- | ------ | -------------- | --------------- |
| 1   | ○   |     |     | id         | 管理区分ID | BIGINT    | -   | ○        |        | AUTO INCREMENT |                 |
| 2   |     |     | ○   | name       | 管理区分名  | VARCHAR   | 50  | ○        |        |                | 例：スーパー管理者、一般管理者 |

| 4   |     |     |     | updated_at | 更新日時   | TIMESTAMP | -   | ○        | now()  |                |                 |

## ポリシー

### PostgreSQL RLS の定義

このプロジェクトでは使用しない（Laravel Policy で対応）

### Laravel Policy の定義


| 操作  | 管理者          | 利用ユーザー | 条件               |
| --- | ------------ | ------ | ---------------- |
| 閲覧  | ○            | ×      | 管理者のみアクセス可能      |
| 登録  | ○（スーパー管理者のみ） | ×      | システム運用で直接DB操作を想定 |
| 編集  | ○（スーパー管理者のみ） | ×      | システム運用で直接DB操作を想定 |
| 削除  | ×            | ×      | システム運用で直接DB操作を想定 |
