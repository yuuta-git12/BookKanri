# 目的
- ユーザーが自分が持っている本をシステムで手軽に管理できるようにする

# 背景
- 手持ちの紙の本、電子書籍が増えてどの本を持っているか分からない
- 電子書籍を持っているのに同じ紙の本を購入することが発生した

# 裏テーマ
- 個人開発を行ったことがないので、挑戦したい
- ポートフォリオとして「動くもの」を作成したい
- モダンなデザインとしたい

# 実現する事項
- 書籍情報を手軽に登録できるようにしたいため、以下のようにする
  - ISBNが登録されている書籍の場合、入力されたISBNを使用して書籍情報を入力する
  - ISBNが存在しない書籍の場合、パンチ入力で対応する

# 用語定義
- ISBN：International Standard Book Number の略。書籍に付与される国際的な識別番号。現在は13桁（ISBN-13）が標準。バーコードや奥付に記載されており、これを使って書籍情報を一意に特定できる。
- 紙の本：物理的な印刷物として存在する書籍
- 電子書籍：デジタルデータとして配信・閲覧する書籍（例：Kindle、楽天Kobo等で購入したもの）
- パンチ入力：キーボード等で手動入力すること

# ロール定義
- 管理者：利用ユーザーの登録・編集・削除・閲覧および、登録データの管理を行える人
- 利用ユーザー：所有している本の登録・編集・削除・閲覧を行える人

# 制約
- 利用ユーザーは他の利用ユーザーの本の内容の閲覧・編集・登録・削除は行えない
- 管理ユーザーはすべての利用ユーザーの本の内容の閲覧・編集・登録・削除を行える

# システム概要
個人が所有する書籍（紙の本・電子書籍）をオンラインで一元管理できるWebアプリケーション。
ISBNを入力することで外部APIから書籍情報を自動取得し、手軽に蔵書を登録・管理できる。
ユーザーごとにデータが独立しており、自分の蔵書のみを管理できる。
管理者は全ユーザーのデータおよびユーザーアカウントを管理できる。

# システム構成

| レイヤー    | 候補技術                                        | 選定理由             |
| ------- | ------------------------------------------- | ---------------- |
| フロントエンド | React                                       |モダン技術による開発のため |
| バックエンド  | Laravel                                     |Inertia.jsによるReactとの連携のため|
| データベース  | PostgreSQL                                  | Laravelとの親和性が高い、Railway上での利用実績が多いため|
| 認証      | メールアドレス＋パスワード認証+2要素認証(google2fa を使用) | Google Authenticatorによる認証コードを行うため |
|スタイル|Tailwind CSS|簡易設計による実装スケジュール削減効果を期待|
| 外部API   | openBDとGoogleBooksAPI                       | 結果がJSON形式で返されるから |
| ホスティング  |Railway（Laravel + PostgreSQL 一体構成）|Inertia.js使用のためフロント・バックを分離しない構成のため、他のサービスと比較し利用コストも低いため|

# 外部連携要件
- ISBNから書籍情報を取得するAPIとの連携
  - openBD：日本の書籍情報をISBNで取得
  - Google Books API：openBDで取得できない場合の補完

# 非機能要件

| 項目     | 要件                               |
| ------ | -------------------------------- |
| 対応デバイス | PC・スマートフォン（レスポンシブデザイン対応）         |
| 対応ブラウザ | Google Chrome 最新版、Safari 最新版     |
| レスポンス  | 通常操作時3秒以内。ISBN検索は外部API依存のため除く    |
| 認証方式   | メールアドレス＋パスワード認証+多要素認証。パスワードはハッシュ化して保存  |

| 通信     | HTTPS必須                          |
| デザイン   | モダンなUI。直感的に操作できること               |
| 可用性    | 個人利用のため厳密なSLAは設けない               |
